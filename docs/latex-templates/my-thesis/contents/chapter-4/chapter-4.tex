\section{Implementasi Smart Contract}

\subsection{LakomiToken (LAK)}

LakomiToken adalah smart contract yang mengimplementasikan standar ERC-20 dengan modifikasi untuk kebutuhan sistem keanggotaan koperasi. Kontrak ini bertanggung jawab atas manajemen keanggotaan melalui mekanisme registry dan pencatatan token simpanan anggota.

Keputusan desain utama pada LakomiToken adalah pemisahan antara status keanggotaan (\textit{membership}) dan token balance. Setiap anggota yang terdaftar memiliki satu suara dalam governance terlepas dari besaran token yang dimilikinya. Pendaftaran anggota dilakukan melalui fungsi \texttt{registerMember()} yang hanya dapat dipanggil oleh akun dengan peran \texttt{MEMBERSHIP_ROLE}.

\begin{lstlisting}[language=Solidity, caption={Fungsi \texttt{getVotingPower} pada LakomiToken}]
function getVotingPower(address account) external view returns (uint256) {
    return isRegisteredMember[account] ? 1 : 0;
}
\end{lstlisting}

Kode di atas menunjukkan implementasi 1-member-1-vote, di mana fungsi \texttt{getVotingPower} mengembalikan nilai 1 jika alamat terdaftar sebagai anggota, dan 0 jika tidak. Pendekatan ini berbeda dengan implementasi DAO konvensional yang umumnya mengembalikan \texttt{balanceOf(account)}.

Selain itu, LakomiToken juga mengimplementasikan mekanisme token locking untuk keperluan collateral pada pinjaman. Token yang dikunci tidak dapat ditransfer namun tetap dihitung sebagai bagian dari total supply, memastikan integritas sistem pada saat penentuan kuorum governance.

\subsection{LakomiVault}

LakomiVault berfungsi sebagai pengelola treasury koperasi, menerima deposit USDC dari anggota dan melacak kepemilikan saham melalui mekanisme \textit{shares}. Kontrak ini juga mengimplementasikan sistem skor kontribusi dan penentuan tingkat reputasi anggota.

Inovasi utama pada LakomiVault adalah implementasi sistem tiering berdasarkan besaran kontribusi anggota. Terdapat tiga tingkat kontribusi:
\begin{itemize}
    \item \textbf{Tier 1} (< 500 USDC): Kontributor tingkat pemula
    \item \textbf{Tier 2} (500--1999 USDC): Kontributor tingkat menengah
    \item \textbf{Tier 3} ($\geq$ 2000 USDC): Kontributor tingkat tinggi
\end{itemize}

\begin{lstlisting}[language=Solidity, caption={Fungsi penentuan tingkat kontribusi}]
function getContributionTier(address member) external view returns (uint8) {
    uint256 deposited = contributions[member];
    if (deposited >= TIER3_THRESHOLD) return 3;
    if (deposited >= TIER2_THRESHOLD) return 2;
    return 1;
}
\end{lstlisting}

Fungsi di atas menentukan tingkat kontribusi berdasarkan total deposit USDC anggota. Tingkat ini kemudian digunakan oleh kontrak LakomiLoans untuk menentukan \textit{Loan-to-Value} (LTV) maksimal yang dapat diakses anggota.

LakomiVault juga mengimplementasikan mekanisme penarikan dengan \textit{timelock} untuk penarikan besar (di atas threshold tertentu). Fitur ini memberikan perlindungan terhadap \textit{rug pull} dan memungkinkan governance untuk meninjau ulang permintaan penarikan yang mencurigakan.

\subsection{LakomiGovern}

LakomiGovern adalah kontrak yang mengimplementasikan mekanisme governance DAO dengan prinsip 1-anggota-1-suara. Kontrak ini memungkinkan anggota terdaftar untuk membuat proposal, melakukan voting, dan mengeksekusi keputusan yang telah disetujui.

Alur proposal governance dimulai dari tahap \textit{Pending}, di mana proposal baru dibuat. Setelah masa voting dimulai, proposal masuk ke tahap \textit{Active} dan anggota dapat memberikan suara (For, Against, atau Abstain). Setelah masa voting berakhir, proposal dievaluasi berdasarkan kuorum dan mayoritas suara. Proposal yang memenuhi kuorum dan mayoritas For masuk ke tahap \textit{Succeeded}, kemudian di-\textit{queue} untuk eksekusi setelah masa timelock.

\begin{lstlisting}[language=Solidity, caption={Mekanisme penentuan kuorum berbasis jumlah anggota}]
function quorum() public view returns (uint256) {
    uint256 members = token.getMemberCount();
    return (members * quorumNumerator) / QUORUM_DENOMINATOR;
}
\end{lstlisting}

Implementasi kuorum di atas berbeda dengan DAO konvensional yang menghitung kuorum berdasarkan persentase dari total supply token. Pada Lakomi, kuorum dihitung berdasarkan persentase dari jumlah anggota terdaftar, memastikan bahwa keputusan governance mencerminkan kehendak mayoritas anggota, bukan mayoritas token.

\subsection{LakomiLoans}

LakomiLoans adalah kontrak yang mengelola fungsi pinjaman darurat dengan sistem LTV bertingkat. Inovasi utama pada kontrak ini adalah integrasi dengan sistem reputasi dari LakomiVault untuk menentukan LTV yang berbeda berdasarkan tingkat kontribusi anggota.

Tabel~\ref{tab:ltv-tiers} menunjukkan skema LTV bertingkat yang diterapkan.

\begin{table}[htbp]
\centering
\caption{Skema LTV Bertingkat}
\label{tab:ltv-tiers}
\begin{tabular}{|c|c|c|}
\hline
\textbf{Tier} & \textbf{Kontribusi} & \textbf{LTV Maksimal} \\
\hline
Tier 1 & < 500 USDC & 30\% \\
\hline
Tier 2 & 500--1999 USDC & 50\% \\
\hline
Tier 3 & $\geq$ 2000 USDC & 70\% \\
\hline
\end{tabular}
\end{table}

\begin{lstlisting}[language=Solidity, caption={Fungsi perhitungan LTV maksimal}]
function _calculateMaxLTV(address borrower) internal view returns (uint256) {
    uint8 tier = vault.getContributionTier(borrower);
    if (tier == 3) return TIER3_LTV;  // 70%
    if (tier == 2) return TIER2_LTV;  // 50%
    return TIER1_LTV;                  // 30%
}
\end{lstlisting}

Skema ini memberikan insentif bagi anggota untuk meningkatkan kontribusinya, karena kontributor tingkat tinggi dapat mengakses pinjaman dengan nilai yang lebih besar relatif terhadap jaminan yang diberikan.

\section{Deployment pada Jaringan DChain}

\subsection{Konfigurasi Deployment}

Deployment smart contract dilakukan dengan urutan sebagai berikut untuk memastikan ketergantungan antar kontrak terpenuhi:
\begin{enumerate}
    \item \textbf{LakomiVault}: Deploy terlebih dahulu sebagai kontrak independen
    \item \textbf{LakomiToken}: Deploy dan konfigurasi referensi ke LakomiVault
    \item \textbf{LakomiGovern}: Deploy dengan referensi ke LakomiToken
    \item \textbf{LakomiLoans}: Deploy terakhir dengan referensi ke LakomiToken dan LakomiVault
\end{enumerate}

Setelah semua kontrak ter-deploy, dilakukan konfigurasi peran (\textit{role grants}) untuk memungkinkan interaksi antar kontrak:
\begin{itemize}
    \item LakomiGovern diberikan \texttt{MEMBERSHIP\_ROLE} pada LakomiToken untuk mendaftarkan anggota
    \item LakomiGovern dan LakomiLoans diberikan \texttt{LOCKER\_ROLE} dan \texttt{BURNER\_ROLE} pada LakomiToken
    \item LakomiGovern diberikan \texttt{GOVERN\_ROLE} pada LakomiVault untuk pengeluaran dana governance
\end{itemize}

\section{Implementasi Frontend}

\subsection{Arsitektur Frontend}

Frontend Lakomi dibangun menggunakan arsitektur web3 traditional dengan stack React + ethers.js. Komunikasi dengan smart contract dilakukan melalui \textit{JSON-RPC} ke node Base Sepolia testnet, dengan transaksi yang ditandatangani oleh dompet Web3 (MetaMask).

\subsection{Fitur Utama}

Frontend menyediakan antarmuka untuk:
\begin{itemize}
    \item Pendaftaran anggota dan manajemen profil
    \item Deposit dan penarikan USDC
    \item Pembuatan dan voting proposal governance
    \item Pengajuan, pencairan, dan pengembalian pinjaman
    \item Dashboard monitoring kontribusi dan reputasi
\end{itemize}

\section{Analisis Keamanan}

\subsection{Access Control}

Seluruh smart contract menggunakan pola \texttt{AccessControl} dari OpenZeppelin untuk manajemen peran berbasis RBAC (\textit{Role-Based Access Control}). Setiap fungsi yang sensitif dilindungi dengan modifier \texttt{onlyRole}, memastikan hanya akun yang berwenang yang dapat memanggil fungsi tersebut.

\begin{lstlisting}[language=Solidity, caption={Contoh implementasi access control}]
function grantRole(bytes32 role, address account)
    public
    onlyRole(getRoleAdmin(role))
{
    _grantRole(role, account);
}
\end{lstlisting}

\subsection{Reentrancy Protection}

Seluruh fungsi yang melakukan transfer token atau eksternal call dilindungi dengan modifier \texttt{nonReentrant} dari OpenZeppelin. Modifier ini memastikan bahwa reentrancy attack tidak dapat dieksploitasi, meskipun hal ini menambah sedikit overhead gas.
